# Security Policy

## Supported Versions

This package is pre-1.0, so only the latest `0.x` release (and the `master`
branch) receives security fixes.

| Version        | Supported          |
| -------------- | ------------------ |
| 0.x (latest)   | :white_check_mark: |
| < latest 0.x   | :x:                |

## Reporting a Vulnerability

Please do **not** open a public issue for security problems.

Report them privately through GitHub's
[private vulnerability reporting](https://github.com/Nejcc/minimax-sdk/security/advisories/new),
or by email to **nejc.cotic@gmail.com**. Include the affected version, a
description, and steps to reproduce.

You can expect an acknowledgement within a few days. If the report is accepted,
a fix and release will follow as soon as practical and you'll be credited in the
advisory unless you prefer otherwise. If it's declined, you'll get an
explanation of why.

Since this SDK talks to a live accounting API, please be careful with
credentials and never include real client secrets, passwords, or tokens in a
report — redact them first.
